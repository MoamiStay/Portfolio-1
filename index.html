COMMUNITY SCIENCE MUSEUM

TECHNOLOGIES

HTML CSS

TASK

Time: 4 weeks
Create a website where the target audience is primary and middle school children and families with young children. The website should be informative, engaging and encourage the viewers to visit the museum itself.

The website should be responsive, attractive and easy to use on a variety of devices.

PROCESS

I started by collecting ideas, colors, fonts and images to create a moodboard.

After finishing the design prototype in Figma I started building the page with HTML. I wrote HTML for all pages before adding any CSS.

PERSONAL EVALUATION

I think I had some good ideas with this project, however the end product fell just a bit short. The sizing and placement on the page makes it feel crowded. The website is also somewhat disorienting.

Although the current content layout needs som adjustments I think the overal idea has good potential.

I am quite satisfied with the color design. I tried to focus on the target audience being a younger age-group and therefore include a lot of color to hopefully create a eyecatching effect and make it more attractive.

GOING FORWARD

Unfortunately I didn't have the chance to get feedback from users in the target age-group. So in the future I aim to base my work more on relevant user-feedback.

I want to further improve my skill in CSS and write more neat code.

I think I would've liked to separate the different sections a bit more clearly and create more room for the page to "breath".

RAINY DAYS

TECHNOLOGIES

HTML CSS

TASK

Time: 2 weeks
Create a functioning website that includes multiple pages and is responsive across multiple devices.

PROCESS

This project was a continuation from a series of previous projects including moodboard, style tile and a prototype in Figma. So with that I could start building the website right away.

I created and wrote all HTML before adding any CSS. I did spend some time trying to figure out how to replicate javascript functionality with plain CSS.

PERSONAL EVALUATION

Some questionable decisions were made. Like making the top line in the header disappear when scrolling in order to save 10px worth of space.

The navigation system in the shop on mobile version had potential but the current version needs work. Taking less space and not cover the other content on the page would be a good start. Preferably using javascript to increase in size and visibility only when interacted with.

Some parts and pages feels unfinished. Hover-effects have poor contrast and content alignment has inconsistencies.

I am quite satisfied with the color and typography design. I wanted the website to have a certain "earthy" feeling to it. Rather than a typical clean and modern look I wished to make the website hold a more intimate and cozy feeling.

GOING FORWARD

I decided to do some big last minute changes and it caused a lot of trouble. So I learned to be careful with adjustments and got better at knowing what to watch out for when altering HTML and CSS.

With Javascript I could pull the product list from an API and create a better solution for sorting products accordingly. I would make the website more smooth by adding animations.

Other than that I would like to further improve upon the points mentioned under Self Assessment.
